## 🔄 Omni Scraper Update Report
*2026-03-17T19:48:09Z*

### 📋 New Changelog Entries (5)
- `2026-03-06` Added a new opus model tier so you can set model: opus in your model's ai_settings to use Claude Opus 4.6. This tier is available through Bedrock and external providers including Anthropic and Snowflake Cortex.
- `2026-03-06` Added support for creating PrivateLink connections to Redshift provisioned clusters. See the Redshift PrivateLink documentation for more information.
- `2026-03-06` Updated the ai_settings.query_all_views_and_fields parameter to accept enabled | disabled | default values instead of true | false . This allows you to control whether the AI queries all views and fields when generating SQL.
- `2026-03-05` Added a new combined view to the My activity page that deduplicates activity by workbook or model and groups entries into time-period sections. You can toggle between the new combined view (default) and the classic view that lists each action in chronological order.
- `2026-03-05` Added the ability to assign a dbt environment to a specific user when creating or editing environments in Omni. If you have permission to manage users, you can assign the environment to any user; otherwise, you can only assign it to yourself or leave it unowned.
